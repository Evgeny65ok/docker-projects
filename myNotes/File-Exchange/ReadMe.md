# Отчёт: Ошибка отображения в файловом обменнике на базе halverneus/static-file-server

## Описание проблемы

При попытке запустить контейнер на базе образа `halverneus/static-file-server` и получить доступ к нему через веб‑браузер наблюдалась следующая ошибка: браузер возвращал пустой ответ (`Empty Response`), несмотря на то, что контейнер имел статус `Up` в системе Docker.

Симптомы:
* контейнер успешно стартовал и отображается в списке запущенных (`docker ps`);
* порт проброшен корректно (например, `0.0.0.0:8080->80/tcp`);
* при обращении по адресу `http://localhost:8080` браузер показывает ошибку «Empty Response» или «No data received»;
* логи контейнера не содержат явных ошибок, указывающих на сбой сервиса.

## Анализ причины возникновения

Основная причина проблемы заключается в особенностях работы образа `halverneus/static-file-server`. В отличие от популярных веб‑серверов (например, Nginx или Apache), данный сервер:

* не генерирует автоматически страницу‑заглушку или листинг директории, если в корневой папке отсутствуют файлы;
* ожидает наличия хотя бы одного статического файла для обслуживания;
* при отсутствии файлов в смонтированной директории просто не формирует HTTP‑ответ, что приводит к разрыву соединения без каких‑либо сообщений.

Корневая директория для обслуживания файлов в образе — `/srv`. Если эта папка пуста (или смонтирована из пустой директории хоста), сервер не может вернуть контент, и соединение завершается с пустым ответом.
