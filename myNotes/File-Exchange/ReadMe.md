# Отчет: Ошибка отображения в файловом обменнике

## Описание проблемы
При запуске образа `halverneus/static-file-server` браузер вернул пустой ответ (Empty Response), хотя контейнер имел статус `Up`.

## Причина
Контейнер был запущен в пустой директории. В отличие от Nginx, данный сервер не генерирует стандартную страницу-заглушку, если в корневой папке (`/srv`) отсутствуют файлы.

## Решение
Добавление любого файла (например, `success.txt`) в рабочую директорию хоста перед монтированием тома решает проблему.
