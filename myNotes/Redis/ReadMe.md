# Отчёт: Развёртывание in‑memory базы данных Redis в контейнеризированной среде

## 1. Запуск контейнера с Redis

Для развёртывания высокопроизводительной in‑memory базы данных Redis был использован официальный оптимизированный образ на базе Alpine Linux — это позволяет сократить размер контейнера и ускорить его запуск без потери функциональности.

Контейнер запущен в фоновом режиме с привязкой к стандартному порту Redis (6379), что обеспечивает совместимость с клиентскими приложениями и библиотеками по умолчанию. Имя контейнера задано в понятной форме для упрощения последующего управления ресурсами в Docker‑среде.

Команда для запуска контейнера:

`docker run -d --name my-redis -p 6379:6379 redis:alpine`

Параметры запуска:
* `-d` — запуск в фоновом (демонизированном) режиме;
* `--name my-redis` — присвоение понятного имени контейнеру;
* `-p 6379:6379` — проброс порта 6379 хоста на порт контейнера для внешнего доступа.

## 2. Взаимодействие с базой данных через Redis CLI

Для проверки работоспособности и выполнения операций с данными использовалась встроенная утилита командной строки Redis CLI (`redis-cli`) — стандартный инструмент администрирования и отладки Redis.

### Основные протестированные операции

1. **Проверка доступности сервера (PING):**
   Выполнена команда `PING`, получен ответ `PONG`. Это подтверждает, что сервер запущен, принимает соединения и готов к обработке запросов.

2. **Запись и чтение строковых значений (SET/GET):**
   Успешно протестированы базовые операции работы с данными:
   * команда `SET <ключ> <значение>` — записано тестовое строковое значение по заданному ключу;
   * команда `GET <ключ>` — выполнено чтение ранее сохранённого значения по тому же ключу, возвращено корректное значение.

   Эти операции демонстрируют базовую функциональность хранилища: возможность сохранять данные в оперативной памяти и оперативно извлекать их по ключу.

## 3. Итоговые выводы

Развёртывание in‑memory базы данных Redis завершено успешно:
* контейнер функционирует в изолированной Docker‑среде;
* порт 6379 доступен для подключения внешних клиентов;
* базовые операции записи и чтения данных выполняются без ошибок;
* высокая скорость отклика при выполнении команд в Redis CLI подтверждает корректную работу движка и отсутствие задержек, характерных для дисковых СУБД.

Полученная конфигурация готова к интеграции в тестовые и продуктивные среды в качестве кэша, брокера сообщений или хранилища сессий. Лёгковесность образа Alpine и изолированность контейнера обеспечивают масштабируемость и безопасность эксплуатации.
