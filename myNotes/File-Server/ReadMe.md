# Отчет: Ошибка развертывания HTTP-сервера для раздачи файлов

## 1. Описание задачи
Настройка контейнера Nginx для раздачи статического файла `test.txt` на порту 8082 с использованием монтирования локальной директории.

## 2. Ход работы и возникшие проблемы
При попытке запуска и проверки сервера возникли следующие ошибки:

### Проблема №1: Ошибка 404 Not Found
При выполнении команды `curl http://localhost:8082/test.txt` сервер вернул ошибку 404.
- **Причина:** В исходном коде и структуре проекта файл был ошибочно назван `tetx.txt` вместо `test.txt`.

### Проблема №2: Некорректное монтирование томов (Volumes)
В браузере вместо содержимого файла отображался список системных директорий (System32).
- **Причина:** Команда `docker run` запускалась из системной папки `C:\Windows\System32`. Переменная `$(pwd)` подставила текущий путь терминала, из-за чего в контейнер пробросились системные файлы Windows вместо файлов проекта.

### Проблема №3: Конфликт имен (Conflict)
При повторных попытках запуска Docker выдавал ошибку: `The container name "/http-server" is already in use`.
- **Причина:** Старый контейнер с тем же именем не был удален перед созданием нового.

## 4. Вывод и решение
Для исправления ситуации необходимо:
1. Переименовать файл в `test.txt`.
2. Перейти в рабочую директорию проекта через `cd` перед запуском Docker.
3. Принудительно удалить старый контейнер командой `docker rm -f http-server`.
