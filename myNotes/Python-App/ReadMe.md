# Отчёт: Запуск Python‑скриптов в Docker — без лишних хлопот

## 1. Подготовка: скрипт на раз‑два

Создали простой файл `script.py` — в нём всего одна команда `print()`, которая выводит приветственное сообщение. Минимум кода, максимум демонстрации возможностей!

## 2. Выполнение кода: Python в контейнере без установки на хост

Запустили скрипт, не устанавливая Python на основную систему: использовали легковесный образ `python:alpine`. Благодаря монтированию тома, локальный файл стал доступен внутри изолированного контейнера.

Команда запуска:

`docker run --rm -v "$(pwd):/app" python:alpine python /app/script.py`

Разбор команды:
* `--rm` — контейнер автоматически удалится после выполнения, не захламляя систему;
* `-v "$(pwd):/app"` — текущая директория хоста монтируется в папку `/app` внутри контейнера;
* `python:alpine` — берём компактный образ Python на базе Alpine Linux (экономим место и ускоряем запуск);
* `python /app/script.py` — указываем интерпретатору, какой файл выполнить.

## 3. Интерактивный режим: Python REPL на лету

Протестировали интерактивный режим (REPL — Read‑Eval‑Print Loop) для быстрой проверки кода прямо в изолированной среде. Это удобно для:
* экспериментов с синтаксисом;
* отладки небольших фрагментов кода;
* проверки работы библиотек без запуска полного скрипта.

Как запустить REPL:

`docker run -it --rm python:alpine python`

После выполнения команды открывается интерактивная сессия Python — можно вводить команды и сразу видеть результат.

## 4. Вывод: почему это удобно

Запуск Python‑скриптов через Docker даёт ряд преимуществ:

* **Независимость от хоста:** не нужно устанавливать Python и зависимости на основной системе — всё работает внутри контейнера.
* **Идентичность среды:** код выполняется в одинаковой среде на любой машине с Docker (Linux, macOS, Windows) — никаких сюрпризов с версиями библиотек.
* **Изоляция:** зависимости и настройки Python не конфликтуют с другими проектами или системными компонентами.
* **Портативность:** образ можно легко передать коллегам или развернуть в продакшене.
* **Гибкость разработки:** монтирование томов позволяет редактировать код в любимом редакторе (например, VS Code) и сразу запускать его в контейнере — цикл разработки становится быстрее и удобнее.

**Итог:** Docker превращает запуск Python‑скриптов в простой и предсказуемый процесс, обеспечивая надёжность и воспроизводимость результатов на разных платформах.
