# Отчёт: Развёртывание уязвимой тестовой среды Metasploitable 2 для отработки навыков кибербезопасности

## 1. Запуск контейнера в изолированной среде

Для проведения учебных тестов по оценке защищённости инфраструктуры был успешно развёрнут контейнер с предустановленной уязвимой системой Metasploitable 2. Запуск выполнялся в интерактивном режиме с присвоением понятного имени контейнеру, что упрощает дальнейшее управление и идентификацию ресурса в среде Docker.

Команда для запуска контейнера:

`docker run --name metasploitable2 -it tleemcjr/metasploitable2`

Такой подход позволяет получить полный доступ к терминалу системы, что необходимо для выполнения диагностических команд и ручного анализа состояния сервисов.

## 2. Первичный анализ целевой системы

Metasploitable 2 представляет собой специально подготовленный образ с намеренно ослабленной безопасностью — он содержит ряд сервисов с известными уязвимостями, что делает его идеальным инструментом для обучения пентестингу и анализа сетевых угроз.

В ходе первичного сканирования были выявлены следующие активно работающие сетевые службы:
* FTP-сервер (порт 21) — с поддержкой анонимного входа;
* Telnet (порт 23) — передача данных в открытом виде, без шифрования;
* HTTP-сервер (порт 80) — веб‑сервисы с известными уязвимостями;
* другие сетевые службы, потенциально доступные для эксплуатации.

Детальное исследование открытых портов и запущенных процессов подтвердило наличие широкого спектра уязвимых компонентов, пригодных для моделирования реальных атак в контролируемой среде.

## 3. Итоговые выводы и перспективы использования

Развёртывание тестовой среды завершено успешно: контейнер Metasploitable 2 функционирует в изолированном окружении Docker, что исключает риск воздействия на рабочую инфраструктуру.

Данный образ является высокоэффективным учебным тренажёром, позволяющим:
* отрабатывать навыки сканирования и выявления уязвимостей;
* изучать методы эксплуатации сетевых сервисов;
* тренировать процедуры реагирования на инциденты;
* закреплять знания по настройке защитных механизмов.

Использование изолированной Docker‑среды обеспечивает безопасный и воспроизводимый процесс обучения, минимизируя риски для реальной инфраструктуры.
